Installation

Get start with Lush this simple steps:


  1. Link CSS

    Add these items to the <head> of your document. This will link the Lush core CSS files into your webpage. 

    <link href="css/lush.animations.min.css" rel="stylesheet" />        
<link href="css/lush.min.css" rel="stylesheet" />

  2. Link JS

    Add this files just before the body close tag <body/>. This will link jQuery, Easing functions and Lush JS core into your webpage. Putting files in the footer makes the page load faster. 


    <script src="js/jquery.js"></script>
<script src="js/jquery.easing.1.3.min.js"></script>
<script src="js/jquery.lush.min.js"></script>

Make sure you are using the right path where you have the downloaded files.

Now you are ready to start using Lush. Go to section Usage to learn how to add markup according to your needs.

Usage

Lush have four usage mode. Let's see each one:


Basic

This mode allows to animate components by separate in a unique container. This mode is usefull for simple content, just like some paragraph and some descriptive images. This mode doesn't use the slider functionality.

Markup

<div class="lush">
  <p data-slide-in="syntax for animation in"
     data-slide-out="syntax for animation out"
     style=""> 
        Some text here to be animated
  </p>
</div>

The data attibutes are described in section syntax. The style attribute is needed to set the initial position of the element, where it will be in inside the lush container once the animation has been completed.

Invocation

Add this line in the footer of your webpage

<script>
  $(document).ready(function(){
    $('.lush').lush();
  });
</script>

This code will invoke the plugin with default properties. For a list of plugin properties see section Properties. Note that the class .lush was added to reference the plugin but is not necessary, the plugin will add it if not present. You can use and id instead of a class to invoke the plugin. The example use a paragraph tag but you can animate any element.

Slider

This mode allows to slide containers with components continuosly.

Markup

<ul class="lush-slider">
  <li class="lush">
    <p data-slide-in="syntax for animation in"
       data-slide-out="syntax for animation out"
       style=""> 
          Some text here to be animated
    </p>
  <li/>
  <li class="lush">
      <img ...>
  <li/>
</ul>

As you can see, now the markup becomes a unordered list where each li tag is container that will animate all elements within. All containers (i.e li tags) will be processed in the order they are declared, from top to bottom.
In this case, the class lush-slider is required for this mode. You can invoke the plugin with an id insted of a class, but the class must be setted.

Invocation

Add this line in the footer of your webpage

<script>
  $(document).ready(function(){
    $('.lush-slider').lush();
  });
</script>

Iframe

This mode is intented to use the slider as a page slider, where each slide will be a page within an iframe that is animated.

Markup

<ul class="lush-slider">
  <li>
    <iframe src="http://" width="NNN" height="NNN"
       data-slide-in="syntax for animation in"
       data-slide-out="syntax for animation out"
       style=""></p>
  <li/>
  <li>
      <iframe ...>
  <li/>
</ul>

Note that the markup is the same but the elements to be animated are iframe with the page to be loaded. In this case the plugin will preload the page, so the animation will begin when the iframe has the page completly loaded. Is recommended set a max-height to each iframe so when first load the page, the container will no take the height of all iframes.

Invocation

As above, just add this lines in the footer of your webpage (Simple, isn't it?)

<script>
  $(document).ready(function(){
    $('.lush-slider').lush({
      manual: true
    });
  });
</script>

In this case, we the set property manual to true so the plugin will animate only in animation, and when the page is completly animated, it will stop. Otherway, the animation will be automatic.

Flexslider

This mode is intended to use with the FlexSlider plugin, that can be downloaded from here. If you are using Flexslider, you probably noticed that the caption are encouraged along with the images, but you can encourage them separately. Here is where Lush comes to the rescue ..

Easily you can add elements to be used as caption and set to them the animation syntax reusing the Flexslider markup.

Markup

<div id="slider" class="flexslider">
  <ul class="slides">
  <li>
    <img src="..." class="ignore" />
    <p data-slide-in="syntax for animation in"
       data-slide-out="syntax for animation out"
       style=""></p>
  <li/>
  <li>
      <img src="..." class="ignore" />
      <p ...>
  <li/>
</div>

As mentioned above, the markup is the recommended by Flexslider, but with a few additions.

First, the class flexslider is required to activate this mode.

Second, the main slider image must have a class ignore. This class will tell the plugin that the element should not been animated.

Note that the animated elements are in same level as the main image, so there is no need to set a caption container.

important Since the slider in this mode is handled and syncronized by Lush Slider, Not all Flexslider options are avaible. If you found some option that you need and not works as expected, please feel free to contact me so I can help you.


Invocation

You can use all Flexslider options with the plugin but the follows are recommended:

<script>
  $(document).ready(function(){
    $('.lush-slider').lush({
      flexslider : {
                    animation : 'slide' // or 'fade'
                  , slideshow: false  // Lush controls the slide timing
                  , useCSS: false     // Avoid conflict with CSS transitions
                  }
    });
  });
</script>

Basically, we hook up the plugin passing to it the Flexslider options. You dont need to invoke Flexslider in the container, Lush will begin automatically to synchronize with Flexslider.
If you are using Flexslider in your project, before to start using it with Lush, copy the options and remove the invocation.

The property useCSS must be setted to false if you that the slide animation jumps abruptly.

The property slideshow with a false value is avoids the Flexslide slides automatically. If yu are using it, make sure you have setted the right slide delay, otherwise Lush animations could be aborted before end.

External Controls

Lush Slider can be manipulated with any element that you want.

Add class lush-external to each element so the plugin can detect the external control.

Add an attribute data-slideto to each control with the slide number to move

For example, if we have a slider with 3 slides, we can add 3 links where each one encourage the slider to a specific slide.

<a class="lush-external" data-slideto="1"> Go Slide 1 </a>
<a class="lush-external" data-slideto="2"> Go Slide 2 </a>
<a class="lush-external" data-slideto="3"> Go Slide 3 </a>

When click on some link, the slide will advaced automatically to the slide indicated by the data attribute.

You can use images, buttons, icons and any clickable element.

Syntax

Lush was designed having in mind two principles:

  • Usability: Lush syntax is semantics, which means that you can describe animation events in the same way you would think it.
  • Maintainable: The animations are based on CSS classes, which makes easier to modify, update and expand the pool of animations.

This section is an introduction guide for the animation syntax.


Input syntax

Animation in syntax must be introduced into the attribute data-slide-in


[at|plus] {DELAY TIME} from {DIRECTION} use {EASING} during {DURATION TIME}

Output syntax

Animation in syntax must be introduced into the attribute data-slide-out


[at|plus] {DELAY TIME} to {DIRECTION} use {EASING} during {DURATION TIME} [force]

Description

The syntax described above has the next interpretation:

In black are the keywords, they are case sensitive and optionals. If you avoid some, the plugin will use a default value.

The sign | represents "or", which means if you use the keyword at you can't use plus, and viceversa.

The unique difference between input and output syntax are the keywords from and to

Output syntax can be omitted, so the plugin will animate the entry of the item and hold its final position without leaving the scene.

Inpput syntax can be omitted but the plugin will animate the elements using the default values for each keyword

Between brackets the are the values (parameters) you must set for the animation, below describe how to fill this values.

Keywords

Name Description Default
at indicates the time in millisecond that the animation should wait before start. 0
plus indicates the time in millisecond that the animation should wait before start. In that case, the time will be calculated adding the value given to the wait time of the precedent animation. If not precedent, it will consider 0. 0
from indicates the animation to be used when the element is ready to step in. See section Animations for a list of animations 'left'
to indicates the animation to be used when the element is ready to step out. See section Animations for a list of animations 'right'
use This is the easing function. The value should be case sensitive. Below you can see a list of available values. More information on easing functions, here 'swing'
during sets the animation duration time. The time the animation will take to move from start to end. 1000
force This options works for animation out only. It forces the slider to make a full animation (in/out) of the element. This option is useful when two elements, in different times, are in the same position, so you will need to force the animation, even if the user pauses the slider. The yellow text in the demo page uses this feature.
This options is activated only adding the word to the timeline, there's no need to add a value for it.		 false

Examples

This example show how to animate a paragraph and the idea involved


Wait, at two seconds, start animation from left and use linear easing during one second.

Then, wait, at three seconds, continue animation to right and use swing easing during one second.

<p data-slide-in="at 2000 from left use linear during 1000"
   data-slide-out="at 3000 to right use swing during 1000"
   style="...">Some text...</p>

Another example using plus and without output animation.


<p data-slide-in="at 2000 from left use linear during 1000"
   style="...">Some text...</p>        
<p data-slide-in="plus 200 from left use linear during 1000"
   style="...">Some text...</p>

The first paragraph will start at 2000 milliseconds (2 seconds exactly).

The second paragraph will start at 2200 milliseconds. 2000 milliseconds from precendent plus 200 millisecond from actual.


Using container elements, like an image link


<a href="..." 
   data-slide-in="at 2000 from left use linear during 1000"
   style="...">
  <img src="..." />        
</a>

The container element will receive the animation and will animate all its content

Events

With events you can take actions when a certain event ocurrs inside the slider

All events are trigger by the container (element with .lush class), so you must hook to it before using the slider.

Event name Description
lushInit Triggers when the slider has been initialized and before to start animations
slideIn Triggers when the slide begin to animate input animations
slideOut Triggers when the slide begin to animate output animations
slideStop Triggers when input animations has been ended. Only fires if property manual is set to true.
slideEnd Triggers when all animations has been ended.

Extending animations

You just need to declare classes in the way the plugin can manipulate to control animations

There are two namespace used to determine if an animation is an input or ouput animation. This namespace are classes .in and .out

To determine state of the animation, you must create a class with the animation name that will be used in the syntax as an initial state, and then a class .live concatenated to same name with the rules for the end state.

All premium animations use the animation property of CSS3. To extend it, is required to add the property animation-fill-mode to the class with the initial state and declare the animation keyframes in file lush.animations.css

With CSS

/* input animation */
.in.[Your animation name] {
  /* animation initial state */
} 
.in.[Your animation name].live {
  /* animation end state */
}
/* output animation */
.out.[Your animation name] {
  /* animation initial state */
} 
.out.[Your animation name].live {
  /* animation end state */
}

With LESS

/* input animation */
.in {
  &.[Your animation name] {
    /* animation initial state */
    &.live {
      /* animation end state */
}}}
/* output animations */
.out {
  &.[Your animation name] {
    /* animation initial state */
    &.live {
      /* animation end state */
}}}

Final notes



Responsive


To work in a responsive enviroment lush slider uses a baseWidth and baseHeight properties to determine the original size of the slider. This size is the same size as your images and text when you build a new slider from scratch.

Lush works in a default mode where the slider size is 1140px width and 450px height. If you create new slide using image an text that doesn't fill well with the default size, you could experience some alignments problems.

Make sure your are building the right way to get the best experience. Anyway, if you find it complex, you can contact me for support, I'll be glad to guide you through the process.

Minified version


A minified version of a javascript or css file, is a compressed version which not include comments, whitespaces and breaklines to reduce the file size and preserve bandwidth. In terms of functionality both versions are absolutely the same. It is recommended to use the compressed version in a production environment. You can identified this minified version with the suffix min . If you need to customize some file, you can minify them with some of the next tools:


  • YUI
  • Closure
  • JS Compress

Lush includes minified files compressed with Yui online compressor.
